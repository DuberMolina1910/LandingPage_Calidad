DISPONIBILIDAD

La disponibilidad hace referencia a la capacidad del sistema para estar disponible, particularmente después de que se produce una falla (Omisión, Caída, Tiempo incorrecto o Respuesta incorrecta). La falla debe ser reconocida y luego el sistema debe responder de alguna manera, debe recuperarse de la misma y como medida de respuesta se especifica el porcentaje de disponibilidad o tiempo para detectar la falla.

INTEROPERABILIDAD

La interoperabilidad es la capacidad de los sistemas para compartir información de manera útil. Es posible que estos sistemas se hayan construido con la intención de intercambiar información o pueden ser sistemas ya existentes. El escenario tiene el objetivo de intercambiar información en donde un sistema hace una solicitud en donde los sistemas se conocen antes del tiempo de ejecución y el resultado es el intercambio de información.

MODIFICABILIDAD

La modificabilidad hace referencia al cambio y al costo en tiempo o dinero de realizar un cambio, incluida la medida en que afecta otros atributos de calidad. Los desarrolladores, instaladores o usuarios finales pueden realizar cambios, y estos cambios deben prepararse. Las tácticas de modificabilidad están enfocadas para preparar los cambios. Estas tácticas para reducir el costo de realizar un cambio incluyen hacer módulos más pequeños, reducir el acoplamiento, ubicar responsabilidades relacionadas y abstraer los servicios comunes.

RENDIMIENTO

El rendimiento se enfoca en la gestión de los recursos del sistema frente a la demanda para lograr un comportamiento de tiempo aceptable, donde se puede medir en términos de rendimiento y latencia para sistemas tanto interactivos como de tiempo real integrados. Este rendimiento se puede mejorar reduciendo la demanda o administrando los recursos de manera más eficiente. La administración más adecuada de los recursos se puede hacer mediante la programación o simplemente aumentando los recursos disponibles.

SEGURIDAD

Los ataques contra un sistema pueden caracterizarse como ataques contra la confidencialidad, integridad o disponibilidad de un sistema o sus datos. La confidencialidad es mantener los datos alejados de los que no deberían tener acceso y dar acceso a los que deberían. La integridad significa que no hay alteraciones no autorizadas de datos, y la disponibilidad significa que el sistema es accesible para aquellos que pueden usarlo.

TESTEABILIDAD

Asegurarse de que un sistema sea fácilmente comprobable trae beneficios tanto del costo de pruebas como de la confiabilidad del sistema. Los arneses de prueba son sistemas de software que tienen recursos de prueba, como algunos casos de prueba e infraestructura para pruebas, por lo que es fácil volver a hacer las pruebas a través de iteraciones. Otro opción es la creación de casos de prueba antes del desarrollo de una funcionalidad, para que los desarrolladores sepan qué deben certificar. Controlar y observar el estado del sistema es una clase importante de tácticas de prueba.

Es una propiedad medible del software que indica que tanto satisface a las partes interesadas.
